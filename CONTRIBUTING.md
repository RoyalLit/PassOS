# Contributing to PassOS

First off, thank you for considering contributing to PassOS! It's people like you who make it a great tool for campus mobility.

## Code of Conduct
By participating in this project, you agree to abide by our [Code of Conduct](./CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs
- Use the [Bug Report Template](.github/ISSUE_TEMPLATE/bug_report.md).
- Check if the bug has already been reported.
- Provide as much detail as possible.

### Suggesting Enhancements
- Use the [Feature Request Template](.github/ISSUE_TEMPLATE/feature_request.md).
- Explain why this enhancement would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Development Setup
Refer to the [Setup Guide](./docs/setup.md) for instructions on how to set up your local development environment.

## Style Guidelines
- Use TypeScript for all new code.
- Follow the existing coding style (ESLint config is provided).
- Write descriptive commit messages.

## Questions?
Feel free to open an issue with the tag `question`.
